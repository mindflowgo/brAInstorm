UA Browser Identity

Proxima uses this browser user-agent for provider views, auth popups, and request header spoofing. Changes apply to future requests immediately, but open tabs may need a reload to fully pick up the new value.

The default stays in the provider catalog. Clear the field or use reset if you want Proxima's bundled Chrome-style identity back.

IM Captured Images

When a provider generates images, Proxima downloads them locally first and returns those local file paths in the MCP response.

Absolute paths are recommended. If left blank, Proxima stores generated images in a temporary local folder and returns those absolute file paths to MCP clients.
